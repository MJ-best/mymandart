## PRD 
Mandarart를 쉽게 만들고 공유하는 앱입니다
기본적으로 로컬에서 작동하며 사용자의 데이터를 Tree 차트 형식으로 기록하여 사용자가 자신의 데이터를 활용하는데 편리하도록 합니다. 

기획의도는 만다라트라고 하는 목표설정과 시각화 방식이 굉장히 유용하지만 동시에 작성하는게 어렵다는 단점이 있었습니다. 총 64개의 액션아이템이 도출되어야 하기 때문에 이를 한번에 하는건 쉽지 않습니다. 그래서 언제든 중단된 곳에서 계속 진행하면서도 만다라트가 가진 강점인 내가 해야하는 액션아이템을 도출하는데에 집중할 수 있도록 하는 것입니다. 

Pain Point 

AS-IS : 만다라트로 계획을 하고 싶어도 어떤 키워드를 잡아야 할지 모름
To-BE : 키워드 추천으로 손쉽게 만들수 있음

AS-IS : 만다라트 기법을 제대로 몰라서 잘 작성하지 못함
To-BE : 한 문장으로 내가 되고 싶은 나를 그린 다음, 이를 위한 8개의 테마를 정하고, 각 테마별로 8개의 액션아이템을 만드는 과정이 UI의 흐름으로 유기적으로 이뤄지며 빈페이지에서 시작하는게 아닌 키워드 예시가 주어지므로 사용자는 UI만 따라가면 만다라트를 만들 수 있음. 

사용자 여정 

1. 한 문장으로 내가 되고 싶은 나를 묘사함. 한 문장을 그냥 적는건 어렵기 때문에 9가지의 키워드가 주어짐 (예 : ㅇㅇ를 ㅇㅇ하게 해내는 ㅇㅇㅇ인 나)
2. 다음 화면으로 자동으로 넘어와서 1번에서 작성한 문장은 맨 위에 항상 있고 8가지의 테마를 하나씩 스크롤하면서 정의함. 키워드가 주어짐
3. 다음으로 넘어와서 각 테마별로 세부 목표를 작성하게함. 단 수치화되어 평가할 수 있는 목록을 작성하게 되며 여기선 체크박스로 작성됨. 
4. 다 완성하면 오른쪽 아래 내 만다라트 보기가 활성화 되어서 언제든 내 만다라트를 볼 수 있고, 이미지로 공유도 가능함
5. 3번 화면에서 세부 목표를 완료하면 (체크박스에 체크표시) 이미지에서도 해당 칸은 색상이 변해서 실행됨을 확인할 수 있게함

## UIUX
3개의 화면전환이 유기적으로 이뤄지며 가로 슬라이드 & 버튼으로 기록하고 공유하는데에 편리하도록 함

### 1번 화면: 나를 정의하는 한 문장 ✍️
*   **목표:** 사용자가 자신의 핵심 목표를 한 문장으로 정의합니다.
*   **UI 구성:**
    *   `TextField`: "내가 되고 싶은 나"를 작성하는 입력 필드.
    *   `키워드 추천 칩/버튼`: 사용자가 쉽게 문장을 구성할 수 있도록 9가지 예시 키워드(예: '능동적으로', '성장하는', '혁신가')를 `Chip` 형태로 제공. 선택 시 입력 필드에 자동 삽입.
    *   `다음` 버튼: 다음 단계로 이동하는 버튼. (Flutter의 `ElevatedButton` 또는 `TextButton`).
*   **상호작용:** 입력 필드에 텍스트가 있을 때만 `다음` 버튼 활성화. 키워드 선택 시 해당 키워드 텍스트가 입력 필드에 추가됩니다.

### 2번 화면: 8가지 핵심 테마 설정 💡
*   **목표:** 정의한 목표를 달성하기 위한 8가지 핵심 테마를 설정합니다.
*   **UI 구성:**
    *   `상단 고정 텍스트`: 1번 화면에서 작성한 "내가 되고 싶은 나" 문장.
    *   `8개의 테마 입력 필드`: 각 테마를 정의하는 `TextField`로 구성된 `ListView` 또는 `Column`. 각 입력 필드는 스크롤 가능하며, 빈 상태로 시작.
    *   `키워드 추천 칩/버튼`: 각 테마별로 연관성 있는 키워드 추천을 `Chip` 형태로 제공.
    *   `다음` 버튼: 다음 단계로 이동하는 버튼.
*   **상호작용:** 모든 8가지 테마 필드에 최소한의 내용이 입력될 때 `다음` 버튼 활성화. 테마 입력 시 실시간으로 로컬 DB에 저장됩니다.

### 3번 화면: 테마별 8가지 세부 액션 아이템 🎯
*   **목표:** 각 테마별로 구체적이고 수치화 가능한 8가지 액션 아이템을 설정하고 관리합니다.
*   **UI 구성:**
    *   `상단 고정 텍스트`: 1번 화면의 목표 문장 및 현재 선택/진행 중인 테마.
    *   `테마별 액션 아이템 리스트`: `CheckboxListTile` 형태로 8개의 세부 목표와 `Checkbox`로 구성. 각 아이템 옆에 `TextField`를 배치하여 세부 목표를 작성.
    *   `만다라트 보기` 버튼: 모든 액션 아이템 작성이 완료되면 화면 우측 하단에 활성화되어, 현재까지의 만다라트 현황을 시각적으로 보여주는 화면으로 이동 (혹은 팝업).
    *   `이미지 공유` 버튼: `만다라트 보기` 화면에서 만다라트 이미지를 공유할 수 있는 버튼.
*   **상호작용:**
    *   `Checkbox` 체크 시: 해당 액션 아이템이 완료된 것으로 표시되며, `만다라트 보기` 화면에서 해당 칸의 색상이 즉시 변경되어 시각적으로 완료를 확인할 수 있습니다.
    *   액션 아이템 작성 시 실시간으로 로컬 DB에 저장됩니다.

### 만다라트 보기 화면 (별도 모달 또는 페이지) 📊
*   **목표:** 완성된 만다라트 구조를 시각적으로 보여주고, 진행 상황을 한눈에 파악하며, 이미지로 공유합니다.
*   **UI 구성:**
    *   `만다라트 그리드 뷰`: 중앙 목표, 8가지 테마, 64가지 액션 아이템이 계층적으로 배치된 그리드 형태의 UI.
    *   `완료 표시`: 체크된 액션 아이템은 다른 색상으로 표시.
    *   `공유 버튼`: 현재 만다라트 뷰를 이미지 파일로 저장하거나 공유할 수 있는 버튼.
*   **상호작용:** 액션 아이템의 체크 상태가 변경될 때마다 이 화면의 시각적 요소가 실시간으로 업데이트됩니다.

## DB
로컬 DB를 활용
로컬DB를 활용하며 각 UI 화면에서 사용자의 입력내용이 캐시되어 자동으로 저장됩니다
데이터 내보내기를 통해 추출 될 때는 Tree구조로 데이터를 보여줌

### 데이터 모델 (React Native 로컬 DB) 🌳
*   **기술 스택:** React Native에서 `react-native-sqlite-storage` (SQLite), `WatermelonDB`, `Realm` 또는 `MMKV`와 같은 로컬 NoSQL 데이터베이스 활용. `WatermelonDB`나 `Realm`은 성능과 스키마 관리 측면에서 유리합니다.
*   **테이블/컬렉션 구조:**
    *   **Goal:**
        *   `id`: UUID (Primary Key)
        *   `centralGoal`: String (중앙 목표 문장)
        *   `createdAt`: DateTime
        *   `updatedAt`: DateTime
    *   **Theme:**
        *   `id`: UUID (Primary Key)
        *   `goalId`: UUID (Goal 테이블의 `id` 참조)
        *   `themeText`: String (8가지 테마 중 하나)
        *   `order`: Integer (테마의 순서 1-8)
        *   `createdAt`: DateTime
        *   `updatedAt`: DateTime
    *   **ActionItem:**
        *   `id`: UUID (Primary Key)
        *   `themeId`: UUID (Theme 테이블의 `id` 참조)
        *   `actionText`: String (각 테마별 8가지 세부 목표)
        *   `isCompleted`: Boolean (체크박스 완료 여부)
        *   `order`: Integer (액션 아이템의 순서 1-8)
        *   `createdAt`: DateTime
        *   `updatedAt`: DateTime
*   **관계:** Goal (1) : (8) Theme : (64) ActionItem의 트리 구조를 가집니다.
*   **데이터 내보내기:**
    *   사용자의 모든 만다라트 데이터를 JSON 또는 YAML 형식의 트리 구조로 추출합니다.
    *   예시:
        <smtcmp_block language="json">
        {
          "goal": "건강하고 활동적인 사람이 되는 나",
          "themes": [
            {
              "themeText": "신체 건강",
              "actionItems": [
                {"actionText": "주 3회 헬스장 방문", "isCompleted": true},
                {"actionText": "매일 30분 이상 유산소 운동", "isCompleted": false},
                // ... 6개 더
              ]
            },
            // ... 7개 더
          ]
        }
        </smtcmp_block>

## Backend
### 기술 스택: Python (Flask/FastAPI 또는 로컬 스크립트) 🐍
본 앱은 로컬에서 작동하는 것을 기본으로 하므로, 전통적인 서버 백엔드는 필수가 아닐 수 있습니다. 하지만 Python을 활용하여 다음과 같은 기능을 구현할 수 있습니다.

*   **데이터 내보내기/보고서 생성 유틸리티:**
    *   **역할:** Flutter 앱에서 내보낸 로컬 DB 파일 (예: JSON)을 읽어들여, 더욱 정교하고 시각적인 보고서를 생성하거나, 다른 형식 (CSV, PDF 등)으로 변환하는 로컬 Python 스크립트.
    *   **구현:** `pandas`를 이용한 데이터 처리, `Jinja2`를 이용한 HTML/PDF 보고서 템플릿 생성, `matplotlib` 또는 `seaborn`을 이용한 데이터 시각화.
*   **키워드 추천 엔진 (선택 사항):**
    *   **역할:** 만약 키워드 추천 로직이 단순한 정적 리스트를 넘어, 사용자 입력 패턴 기반의 동적인 추천이나 자연어 처리(NLP)를 포함한다면, Python의 `Flask` 또는 `FastAPI`를 이용한 로컬 API 서버를 구축하여 이 기능을 제공할 수 있습니다.
    *   **구현:** `scikit-learn`이나 `Spacy` 같은 NLP 라이브러리를 활용하여 텍스트 유사도 기반의 키워드 추천 로직 구현.
*   **데이터 백업 및 복원 스크립트:**
    *   **역할:** 로컬 DB 파일의 자동 백업 및 특정 시점으로의 복원을 돕는 스크립트.
    *   **구현:** 파일 시스템 접근 및 압축 라이브러리 활용.

## Frontend
### 기술 스택: Flutter 🚀
*   **아키텍처:** `Provider`, `Riverpod`, 또는 `BLoC`과 같은 상태 관리 패턴을 사용하여 앱의 복잡성을 관리하고 예측 가능한 상태 변화를 만듭니다.
*   **주요 라이브러리/패키지:**
    *   `sqflite` (또는 `hive`, `isar`): 로컬 데이터베이스 연동.
    *   `shared_preferences`: 간단한 사용자 설정 저장 (예: 마지막 방문 페이지).
    *   `go_router` (또는 `auto_route`): 선언적 라우팅을 통한 화면 간 유기적 전환 관리.
    *   `flutter_slidable`: 화면 2, 3에서 각 아이템에 대한 스와이프 액션 (수정/삭제) 구현.
    *   `image_picker` / `path_provider`: 이미지 공유 시 앱 내부 스크린샷 저장 및 파일 시스템 접근.
    *   `share_plus`: 생성된 만다라트 이미지를 외부 앱으로 공유.
    *   `flutter_riverpod` (혹은 `provider`): 효과적인 상태 관리.
    *   `charts_flutter` (혹은 `syncfusion_flutter_charts`): 만다라트 보기 화면에서 데이터를 더욱 시각적으로 표현할 수 있는 차트 구성.
*   **주요 기능 구현:**
    *   **반응형 UI:** 다양한 기기 크기 (모바일, 태블릿)에 맞춰 UI가 유연하게 조정되도록 구현.
    *   **실시간 저장 및 로딩:** 사용자 입력(`TextField`의 `onChanged` 리스너 활용)이 발생할 때마다 로컬 DB에 데이터를 즉시 저장하고, 앱 재시작 시 중단된 지점부터 데이터를 로드하여 이어서 작업할 수 있도록 구현.
    *   **키워드 추천:** `Chip` 위젯을 사용하여 시각적으로 명확하고 선택하기 쉬운 키워드 추천 UI 구현.
    *   **만다라트 시각화:** `CustomPaint` 위젯을 사용하여 중앙 목표, 테마, 액션 아이템을 트리/그리드 형태로 시각화하고, `AnimatedContainer` 등을 활용하여 완료된 액션 아이템의 색상 변화를 부드럽게 표현.
    *   **이미지 공유:** `RenderRepaintBoundary`를 사용하여 특정 위젯(만다라트 뷰)을 이미지로 캡처하고, `share_plus` 패키지를 통해 공유 기능 구현.
    *   **데이터 내보내기:** 앱 내에서 현재 만다라트 데이터를 JSON/YAML 파일로 내보내는 기능 구현.

## Backend
### 기술 스택: Python (Flask/FastAPI 또는 로컬 스크립트) 🐍
본 앱은 로컬에서 작동하는 것을 기본으로 하므로, 전통적인 서버 백엔드는 필수가 아닐 수 있습니다. 하지만 Python을 활용하여 다음과 같은 기능을 구현할 수 있습니다.

*   **데이터 내보내기/보고서 생성 유틸리티:**
    *   **역할:** Flutter 앱에서 내보낸 로컬 DB 파일 (예: JSON)을 읽어들여, 더욱 정교하고 시각적인 보고서를 생성하거나, 다른 형식 (CSV, PDF 등)으로 변환하는 로컬 Python 스크립트.
    *   **구현:** `pandas`를 이용한 데이터 처리, `Jinja2`를 이용한 HTML/PDF 보고서 템플릿 생성, `matplotlib` 또는 `seaborn`을 이용한 데이터 시각화.
*   **키워드 추천 엔진 (선택 사항):**
    *   **역할:** 만약 키워드 추천 로직이 단순한 정적 리스트를 넘어, 사용자 입력 패턴 기반의 동적인 추천이나 자연어 처리(NLP)를 포함한다면, Python의 `Flask` 또는 `FastAPI`를 이용한 로컬 API 서버를 구축하여 이 기능을 제공할 수 있습니다.
    *   **구현:** `scikit-learn`이나 `Spacy` 같은 NLP 라이브러리를 활용하여 텍스트 유사도 기반의 키워드 추천 로직 구현.
*   **데이터 백업 및 복원 스크립트:**
    *   **역할:** 로컬 DB 파일의 자동 백업 및 특정 시점으로의 복원을 돕는 스크립트.
    *   **구현:** 파일 시스템 접근 및 압축 라이브러리 활용.

## Frontend
### 기술 스택: React Native ⚛️
*   **아키텍처:** `Context API`, `Redux`, `Zustand`, `Recoil` 또는 `MobX`와 같은 상태 관리 패턴을 사용하여 앱의 복잡성을 관리하고 예측 가능한 상태 변화를 만듭니다.
*   **주요 라이브러리/패키지:**
    *   `react-native-sqlite-storage` (또는 `WatermelonDB`, `Realm`): 로컬 데이터베이스 연동.
    *   `@react-native-async-storage/async-storage`: 간단한 사용자 설정 저장 (예: 마지막 방문 페이지).
    *   `React Navigation`: 선언적 라우팅을 통한 화면 간 유기적 전환 관리.
    *   `react-native-swipe-list-view` (또는 `Gesture Handler`와 `Reanimated`를 활용한 커스텀 구현): 화면 2, 3에서 각 아이템에 대한 스와이프 액션 (수정/삭제) 구현.
    *   `react-native-image-picker` / `react-native-fs`: 이미지 공유 시 앱 내부 스크린샷 저장 및 파일 시스템 접근.
    *   `react-native-share`: 생성된 만다라트 이미지를 외부 앱으로 공유.
    *   `Redux` (혹은 `Zustand`, `Context API`): 효과적인 상태 관리.
    *   `react-native-chart-kit` (혹은 `victory-native`, `Recharts`): 만다라트 보기 화면에서 데이터를 더욱 시각적으로 표현할 수 있는 차트 구성.
*   **주요 기능 구현:**
    *   **반응형 UI:** 다양한 기기 크기 (모바일, 태블릿)에 맞춰 UI가 유연하게 조정되도록 구현.
    *   **실시간 저장 및 로딩:** 사용자 입력(`TextInput`의 `onChangeText` 프롭 활용)이 발생할 때마다 로컬 DB에 데이터를 즉시 저장하고, 앱 재시작 시 중단된 지점부터 데이터를 로드하여 이어서 작업할 수 있도록 구현.
    *   **키워드 추천:** `Chip` 위젯을 사용하여 시각적으로 명확하고 선택하기 쉬운 키워드 추천 UI 구현.
    *   **만다라트 시각화:** `react-native-svg` 라이브러리 또는 커스텀 컴포넌트를 사용하여 중앙 목표, 테마, 액션 아이템을 트리/그리드 형태로 시각화하고, React Native의 `Animated API` 또는 `Reanimated` 라이브러리를 활용하여 완료된 액션 아이템의 색상 변화를 부드럽게 표현.
    *   **이미지 공유:** `react-native-view-shot` 라이브러리를 사용하여 특정 뷰(만다라트 뷰)를 이미지로 캡처하고, `react-native-share` 패키지를 통해 공유 기능 구현.
    *   **데이터 내보내기:** 앱 내에서 현재 만다라트 데이터를 JSON/YAML 파일로 내보내는 기능 구현.
